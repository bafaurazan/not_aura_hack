# not_aura — dokumentacja zespołowa

Zbiór zasad i praktyk pracy z kodem, ROS2 oraz systemem kontroli wersji w projekcie `not_aura`.

## Zakres dokumentacji

Repozytorium zawiera trzy główne obszary:

- `version_control` — zasady pracy z gitem, branchami, commitami i code review,
- `code` — wskazówki dotyczące stylu kodu, OOP i przydatnych narzędzi,
- `ros` — konwencje tworzenia paczek ROS2 i organizacji workspace.

## Jak korzystać z tej dokumentacji

- Traktuj te pliki jako punkt odniesienia przy codziennej pracy.
- Przy zmianie zasad aktualizuj odpowiedni plik w katalogu tematycznym.
- Dla nowych paczek ROS2 wykorzystuj szablon: [ros/ros_readme.md](ros/ros_readme.md).
